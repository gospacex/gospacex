# JWT 中间件自动生成工具 - 需求规格说明书

## 一、项目概述

### 1.1 目标
开发一个代码生成工具（`gpx micro-bff`），能够根据用户输入的参数，自动在目标项目中生成完整的 JWT 中间件代码，包括通用封装层和框架适配层。

### 1.2 核心命令

| 命令 | 说明 |
|------|------|
| `gpx micro-bff --name myshop --http gin --middleware jwt` | 生成 Gin 框架的 JWT 中间件 |
| `gpx micro-bff --name myshop --http hertz --middleware jwt` | 生成 Hertz 框架的 JWT 中间件 |

### 1.3 设计原则
- **分层清晰**：pkg 层提供通用能力，internal/middleware 层提供框架适配
- **框架解耦**：pkg/jwt 不依赖任何 HTTP/RPC 框架
- **可扩展**：支持后续增加更多中间件类型和框架

## 二、BFF 层中间件完整规划

BFF（Backend For Frontend）层中间件面向**客户端请求**，关注用户体验和边缘安全。

### 2.1 认证与鉴权类

| 中间件 | 功能描述 | 实现要点 |
|--------|----------|----------|
| **JWT** | 基于 JSON Web Token 的用户身份验证 | Token 生成、解析、刷新；支持 Header/Cookie/Query 多种传输方式 |
| **Basic Auth** | HTTP 基本认证 | 验证用户名/密码；支持 Base64 解码；可与用户系统集成 |
| **API Key** | API 密钥认证 | 支持 Header 或 Query 参数传递；支持密钥轮换和权限分级 |

### 2.2 限流类

| 中间件 | 功能描述 | 算法/策略 |
|--------|----------|-----------|
| **令牌桶** | 允许一定程度的突发流量 | 固定速率生成令牌；支持容量配置 |
| **滑动窗口** | 精确控制时间窗口内的请求数 | 基于时间分片；内存/Redis 存储 |
| **固定窗口** | 简单的时间段计数 | 实现简单，可能有边界双倍请求问题 |

**限流维度**：
- 按用户 ID
- 按 IP 地址
- 按 API 路径
- 按请求方法

### 2.3 黑白名单类

| 中间件 | 功能描述 | 匹配规则 |
|--------|----------|----------|
| **IP 白名单** | 只允许特定 IP 访问 | 支持单 IP、IP 段、CIDR 格式 |
| **IP 黑名单** | 禁止特定 IP 访问 | 支持动态更新；性能考虑使用布隆过滤器 |
| **用户白名单** | 特定用户绕过限制 | 用于内部测试或 VIP 用户 |

### 2.4 安全防护类

| 中间件 | 功能描述 | 防护目标 |
|--------|----------|----------|
| **CORS** | 跨域资源共享配置 | 浏览器跨域请求限制 |
| **CSRF** | 跨站请求伪造防护 | Token 验证；Referer/Origin 检查 |
| **请求签名** | 请求完整性验证 | 防篡改、防重放（时间戳+Nonce） |
| **请求加密** | 请求体加密传输 | 敏感数据保护 |

### 2.5 可观测性类

| 中间件 | 功能描述 | 采集数据 |
|--------|----------|----------|
| **链路追踪** | 分布式请求追踪 | 自动注入/提取 TraceID、SpanID |
| **Metrics 采集** | 请求指标统计 | QPS、延迟分布、错误率、活跃连接数 |
| **访问日志** | 请求响应记录 | 请求方法、路径、状态码、耗时、客户端信息 |

### 2.6 性能优化类

| 中间件 | 功能描述 | 优化手段 |
|--------|----------|----------|
| **Gzip 压缩** | 响应体压缩 | 根据 Accept-Encoding 自动选择压缩算法 |
| **请求缓存** | 缓存 GET 请求响应 | 支持 Cache-Control；基于 URL+Query 的缓存键 |
| **响应合并** | 合并多个小响应 | Server Push 或批处理接口 |
| **超时控制** | 请求级超时 | 可配置不同接口的超时时间 |

### 2.7 路由与转发类

| 中间件 | 功能描述 | 应用场景 |
|--------|----------|----------|
| **路径重写** | URL 路径转换 | 版本路由、旧接口兼容 |
| **子域名路由** | 根据子域名分发 | 多租户架构 |
| **请求转发** | 动态修改上游目标 | 灰度发布、A/B 测试 |
| **健康检查** | 服务健康状态响应 | 负载均衡探测 |

### 2.8 BFF 层中间件优先级

| 优先级 | 中间件类型 | 说明 |
|--------|------------|------|
| **P0（必须）** | CORS、链路追踪、访问日志 | 基础设施必备 |
| **P1（高）** | JWT、限流、Gzip | 安全和性能基础 |
| **P2（中）** | IP 黑白名单、请求签名、超时控制 | 按需启用 |
| **P3（低）** | CSRF、请求缓存、子域名路由 | 特殊场景 |

## 三、SRV 层中间件完整规划

SRV（Service）层中间件面向**内部服务调用**，关注服务治理和数据一致性。

### 3.1 服务治理类

#### 限流与熔断

| 拦截器 | 功能描述 | 实现细节 |
|--------|----------|----------|
| **服务端限流** | 保护服务不被特定消费者压垮 | 按调用方服务名/AppKey 差异化限流；支持方法级 QPS 限制 |
| **熔断降级** | 防止级联故障扩散 | 错误率阈值熔断（如 50%）；慢调用比例熔断（RT > 500ms 超 30%）；降级策略（缓存、默认值、空响应） |
| **舱壁隔离** | 资源隔离防止相互影响 | 按服务/方法分配独立线程池或连接池 |

#### 重试与超时

| 拦截器 | 功能描述 | 实现细节 |
|--------|----------|----------|
| **幂等重试** | 处理临时性网络故障 | 仅限幂等方法（GET、PUT、DELETE）；支持指数退避；最大重试次数配置 |
| **超时控制** | 避免慢调用阻塞 | 方法级超时配置；超时从 BFF 向下游传播 |

### 3.2 安全与审计类

#### 内网认证

| 拦截器 | 功能描述 | 认证方式 |
|--------|----------|----------|
| **mTLS** | 双向证书校验 | 基于证书的服务身份验证 |
| **服务间 JWT** | 透传用户身份 | 验证内网 JWT，不重新认证用户 |
| **Service Account** | 服务身份识别 | 基于服务名/命名空间/K8s ServiceAccount |
| **API Key 验证** | 内部调用密钥 | 要求调用方携带有效 Key |

#### 审计日志

| 拦截器 | 功能描述 | 记录内容 |
|--------|----------|----------|
| **全量请求审计** | 记录每次 RPC 调用 | 调用方服务名、方法名、参数（脱敏）、结果、耗时、时间戳 |
| **敏感操作审计** | 数据变更强制记录 | 写操作（Create/Update/Delete）必须记录完整上下文 |
| **异常调用审计** | 失败请求追踪 | 错误码、错误堆栈、调用链 |

### 3.3 可观测性类

#### 深度 Metrics

| 拦截器 | 功能描述 | 指标类型 |
|--------|----------|----------|
| **RED 监控** | 服务黄金指标 | Rate（QPS）、Errors（错误数/率）、Duration（P50/P95/P99 延迟） |
| **业务指标** | 业务自定义监控 | 订单量、用户注册量、库存变化等 |
| **客户端标识** | 标记调用来源 | 调用方服务名、IP、版本号、环境（prod/staging） |

#### 日志增强

| 拦截器 | 功能描述 | 增强内容 |
|--------|----------|----------|
| **链路标识注入** | 关联分布式日志 | TraceID、SpanID、ParentSpanID |
| **采样日志** | 控制日志量 | 慢请求全量记录；正常请求按比例采样 |
| **错误堆栈增强** | 完整错误上下文 | gRPC status + 业务错误码 + 调用链 |

#### 分布式链路追踪

| 拦截器 | 功能描述 | 实现要点 |
|--------|----------|----------|
| **OTel 集成** | 标准可观测性 | 自动创建 Span；Context 传递；对接 Jaeger/Zipkin |
| **业务 Span 标注** | 增加业务语义 | 添加订单号、用户 ID、商品 ID 等业务字段 |
| **跨服务边界标识** | 区分调用边界 | 标记客户端/服务端 Span 类型 |

### 3.4 数据转换与校验类

#### 请求预处理

| 拦截器 | 功能描述 | 处理内容 |
|--------|----------|----------|
| **参数校验** | 请求数据有效性 | 复用 Protobuf validate 规则；必填字段、格式校验（邮箱/手机）、范围校验 |
| **默认值填充** | 补齐缺失字段 | 为可选字段提供业务默认值 |
| **数据脱敏** | 敏感信息保护 | 日志、Metrics 中自动脱敏密码、身份证、手机号 |

#### 响应后处理

| 拦截器 | 功能描述 | 处理内容 |
|--------|----------|----------|
| **错误码转换** | 统一错误响应 | 内部业务错误码 → 标准 gRPC status（如 1001 → InvalidArgument） |
| **敏感字段过滤** | 按权限返回 | 根据调用方角色过滤敏感字段（如管理员可见手机号） |
| **大字段裁剪** | 减少传输量 | 自动移除不必要的冗余信息（如日志中的 debug 字段） |

### 3.5 流量治理类

#### 路由与染色

| 拦截器 | 功能描述 | 路由策略 |
|--------|----------|----------|
| **灰度发布** | 流量按比例/规则路由 | 基于 Header（x-canary）、用户 ID 哈希、Cookie 路由到灰度实例 |
| **蓝绿部署** | 环境隔离 | 通过 metadata 标识环境（env: blue/green） |
| **全链路压测** | 压测流量特殊处理 | 识别压测标记 Header（x-peach-test），穿透到下游，写入影子表/队列 |

#### 优先级与调度

| 拦截器 | 功能描述 | 调度策略 |
|--------|----------|----------|
| **优先级队列** | 核心业务优先处理 | 根据业务类型分配优先级（高/中/低） |
| **请求合并** | 批量处理相同请求 | 短时间窗口内相同参数请求合并（如热点商品查询） |
| **公平调度** | 防止单消费者占满 | 按调用方分配配额；Fair Queue 算法 |

### 3.6 数据库访问层拦截器

#### 连接管理

| 拦截器 | 功能描述 | 管理内容 |
|--------|----------|----------|
| **连接池监控** | 连接健康度 | 空闲连接、活跃连接、等待队列长度、获取连接耗时 |
| **读写分离** | 流量路由 | SELECT 走从库；INSERT/UPDATE/DELETE 走主库 |
| **连接预热** | 启动时建立连接 | 避免冷启动慢查询 |

#### SQL 治理

| 拦截器 | 功能描述 | 治理措施 |
|--------|----------|----------|
| **慢 SQL 拦截** | 识别性能问题 | 超时阈值（如 100ms）告警 + 日志；自动采集执行计划 |
| **SQL 注入防护** | 安全拦截 | 参数化查询校验；危险关键字检测 |
| **分库分表路由** | 数据分片 | 根据 Sharding Key（如 user_id % 64）路由到具体表/库 |

#### 事务管理

| 拦截器 | 功能描述 | 事务控制 |
|--------|----------|----------|
| **声明式事务** | 简化事务代码 | 自动开启/提交/回滚事务 |
| **传播行为控制** | 事务边界管理 | REQUIRED（默认）、REQUIRES_NEW、NESTED、SUPPORTS 等 |

### 3.7 缓存策略类

| 拦截器 | 功能描述 | 缓存策略 |
|--------|----------|----------|
| **多级缓存** | L1 本地 + L2 Redis | 自动穿透、回填；支持缓存预热 |
| **防缓存击穿** | 热点数据保护 | 互斥锁（singleflight）防止大量请求同时打到 DB |
| **防缓存穿透** | 不存在的数据拦截 | 布隆过滤器快速判断不存在，直接拒绝 |
| **缓存异步刷新** | 避免阻塞 | 异步更新即将过期的热点数据 |

### 3.8 SRV 层拦截器优先级

| 优先级 | 拦截器类型 | 说明 |
|--------|------------|------|
| **P0（必须）** | 链路追踪、审计日志、服务间认证 | 可观测性和安全基础 |
| **P1（高）** | 限流/熔断、Metrics、参数校验、超时控制 | 服务稳定性保障 |
| **P2（中）** | 灰度路由、SQL 治理、缓存拦截、幂等重试 | 业务能力和性能优化 |
| **P3（低）** | 请求合并、优先级调度、全链路压测 | 高级场景按需使用 |

## 四、BFF 与 SRV 层中间件对比

| 维度 | BFF 层 | SRV 层 |
|------|--------|--------|
| **服务对象** | 客户端（浏览器/App） | 内部服务 |
| **认证方式** | JWT、OAuth、Basic Auth | mTLS、Service Account、API Key |
| **限流维度** | 用户 ID、IP、API 路径 | 调用方服务名、方法名 |
| **CORS/CSRF** | ✅ 需要处理 | ❌ 内部网络，不需要 |
| **审计内容** | 用户行为日志 | 服务调用链审计 |
| **流量路由** | 入口染色、A/B 测试 | 灰度发布、蓝绿部署 |
| **数据校验** | 格式校验、必填检查 | 深度业务校验、数据脱敏 |
| **重试策略** | 客户端重试（用户感知） | 服务端幂等重试（后台静默） |
| **压缩** | HTTP 层 gzip | 传输层压缩（可选） |
| **缓存** | 响应缓存 | 数据库缓存、多级缓存 |
| **事务处理** | 无（无数据库直接操作） | 声明式事务、分库分表 |

## 五、框架适配策略

### 5.1 BFF 层框架适配

| HTTP 引擎 | 中间件形式 | 适配要点 |
|-----------|------------|----------|
| **Gin** | `gin.HandlerFunc` | 使用 `c.GetHeader()` / `c.Set()` / `c.JSON()` |
| **Hertz** | `app.HandlerFunc` | 使用 `c.GetHeader()` / `c.Set()` / `c.JSON()` |
| **Echo**（规划） | `echo.MiddlewareFunc` | 使用 `c.Request().Header` / `c.Set()` |
| **Fiber**（规划） | `fiber.Handler` | 使用 `c.Get()` / `c.Locals()` |

### 5.2 SRV 层框架适配

| RPC 引擎 | 拦截器形式 | 适配要点 |
|----------|------------|----------|
| **gRPC** | `grpc.UnaryServerInterceptor` | 从 `metadata` 提取信息；使用 `status.Error` 返回错误 |
| **Kitex** | `endpoint.Middleware` | 从 `rpcinfo` 获取调用信息；使用 `kerrors` 返回错误 |
| **Dubbo-go**（规划） | `filter.Filter` | 适配 Dubbo 的 Filter 接口 |

## 六、代码生成输出规划

### 6.1 目录结构（以 Gin 为例）

```
myshop/
├── pkg/
│   └── jwt/                    # 通用 JWT 包（所有项目共用）
│       ├── jwt.go              # New, GenerateToken, ParseToken, RefreshToken
│       ├── options.go          # WithXXX 配置函数
│       └── types.go            # Config, CustomClaims 定义
├── internal/
│   └── middleware/
│       └── jwt.go              # JWTMiddleware, RefreshTokenHandler（Gin 版本）
└── cmd/server/
    └── main.go                 # 集成示例（包含 jwtService 初始化和路由挂载）
```

### 6.2 生成文件说明

| 文件路径 | 触发条件 | 内容说明 |
|----------|----------|----------|
| `pkg/jwt/*.go` | 总是生成 | 核心 JWT 逻辑，与框架无关 |
| `internal/middleware/jwt.go` | `--http gin` | Gin 版本的中间件 |
| `internal/middleware/jwt_hertz.go` | `--http hertz` | Hertz 版本的中间件 |
| `cmd/server/main.go` | 总是更新 | 添加 JWT 初始化、示例路由、刷新接口 |

### 6.3 后续扩展

- **多中间件支持**：`--middleware jwt,cors,ratelimit`
- **SRV 层支持**：`gpx micro-srv --name userservice --rpc grpc --middleware jwt,ratelimit`
- **配置文件生成**：支持从配置文件读取 JWT 配置（`configs/jwt.yaml`）

## 八、验收标准

### 8.1 功能验收
- [ ] Token 生成：能正常生成符合 JWT 规范的 Token
- [ ] Token 解析：能正确解析合法 Token 并提取 Claims
- [ ] Token 过期：过期 Token 解析失败
- [ ] Token 刷新：满足条件的 Token 能成功刷新
- [ ] 中间件集成：正确拦截请求并验证 Token
- [ ] 错误响应：缺失/无效 Token 返回标准 401

### 8.2 框架适配验收
- [ ] Gin：中间件符合 `gin.HandlerFunc` 签名，使用 `gin.Context`
- [ ] Hertz：中间件符合 `app.HandlerFunc` 签名，使用 `app.RequestContext`

### 8.3 代码规范验收
- [ ] pkg/jwt 不引入任何 HTTP/RPC 框架依赖
- [ ] 函数式选项模式正确实现
- [ ] 包含必要的注释和示例
- [ ] 错误处理完整，不吞没错误
