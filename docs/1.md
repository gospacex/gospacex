# GPX 脚手架 - 完整需求文档

## 一、产品定位

**GPX** 是一个**多范式微服务脚手架生成工具**，支持生成网关、微服务、单体应用、脚本工具和 AI Agent 五种项目类型，内置多种技术栈选型，提供从项目初始化到代码生成的完整解决方案。

### 1.1 支持的项目类型

| 命令 | 类型 | 说明 |
| :--- | :--- | :--- |
| `gpx gateway` | 网关 | 生成 API 网关项目 |
| `gpx micro` | 微服务 | 生成完整微服务项目（BFF + 多个微服务） |
| `gpx monolith` | 单体 | 生成单体应用 |
| `gpx script` | 脚本 | 生成工具脚本项目 |
| `gpx agent` | AI Agent | 生成智能体项目 |

### 1.2 技术栈概览

| 层次 | 默认技术 | 可选技术                          |
| :--- | :--- |:------------------------------|
| BFF HTTP 框架 | Gin | Hertz                         |
| 微服务通信协议 | gRPC | Kitex                         |
| IDL 描述语言 | Protobuf | Thrift                        |
| 服务注册中心 | 直连（无） | Consul、Etcd、Nacos、Polaris     |
| 配置中心 | 本地 YAML | vipper、Nacos、Apollo |


## 二、核心命令结构

### 2.1 主命令格式

```bash
gpx [command] [options]
```

### 2.2 micro 命令详解

`gpx micro` 是核心命令，用于一键生成包含 **BFF 层**和多个**微服务**的完整 Go 项目骨架。

#### 快速开始

```bash
# 交互式模式（无参数）
gpx micro

# 命令行模式
gpx micro --name myshop --output ./output --bff h5 --modules product,order
```

#### 参数列表

**必填参数**

| 参数 | 说明 | 示例 |
| :--- | :--- | :--- |
| `--name` | 微应用名称（项目名） | `--name myShop` |
| `--output` / `-o` | 项目输出目录 | `--output ./output` |
| `--bff` | BFF 层名称 | `--bff h5` |
| `--modules` | 微服务列表，逗号分隔 | `--modules product,order` |

**架构与协议参数**

| 参数 | 默认值 | 说明 | 可选值 |
| :--- | :--- | :--- | :--- |
| `--style` | `standard` | 微服务架构风格 | `standard`、`ddd` |
| `--mesh` | `none` | 服务网格 | `none`、`istio`、`linkerd` |
| `--idl` | `proto` | IDL 描述语言 | `proto`、`thrift` |
| `--protocol` | `grpc` | 微服务通信协议 | `grpc`、`kitex` |
| `--http` | `gin` | BFF HTTP 框架 | `gin`、`hertz` |

**服务治理参数**

| 参数 | 默认值 | 说明 | 可选值 |
| :--- | :--- | :--- | :--- |
| `--register` | —（直连） | 服务注册中心 | `consul`、`etcd`、`nacos`、`polaris` |
| `--config` | —（本地 YAML） | 配置中心 | `nacos`、`apollo`、`consul` |
| `--trace` | — | 调用链追踪 | `jaeger`、`skywalking`、`zipkin` |

**数据库参数（反向工程）**

| 参数 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `--db-host` | `127.0.0.1` | 数据库主机 |
| `--db-port` | `3306` | 数据库端口 |
| `--db-user` | `root` | 数据库用户名 |
| `--db-password` | `123456` | 数据库密码 |
| `--db-name` | `myshop` | 数据库名 |
| `--db-table` | — | 表名，逗号分隔多表 |

**联表查询参数**

| 参数 | 说明 | 格式 |
| :--- | :--- | :--- |
| `--db-join-condition` / `--djc` | 联表条件（可多次指定） | `table1.field1=table2.field2` |
| `--db-join-style` / `--djs` | 联表关系类型（与 condition 一一对应） | `table1:table2=1tn` |

联表关系类型说明：

| 值 | 含义 |
| :--- | :--- |
| `1t1` | 一对一 |
| `1tn` | 一对多 |
| `nt1` | 多对一 |
| `ntn` | 多对多 |

**功能开关参数**

| 参数 | 默认值 | 说明 |
| :--- | :--- | :--- |
| `--middleware` | `false` | 生成中间件（BFF）/拦截器（SRV） |
| `--test` | `false` | 生成接口测试代码和 Shell 脚本 |
| `--config-file` | — | 从配置文件读取参数（支持 YAML/JSON/TOML） |


## 三、微服务架构设计

### 3.1 架构风格说明

| 风格 | 适用场景 | 代码组织特点 |
| :--- | :--- | :--- |
| **standard** | 标准微服务（CRUD 为主） | `service/`、`dao/`、`model/` 分层 |
| **ddd** | 领域驱动设计微服务 | `application/`、`domain/`、`infra/` 分层 |

### 3.2 服务网格支持

通过 `--mesh` 参数独立选择，与架构风格正交：

| 选项 | 说明 |
| :--- | :--- |
| `none` | 不启用服务网格 |
| `istio` | 生成 Istio VirtualService、DestinationRule 等配置 |
| `linkerd` | 生成 Linkerd 服务配置文件 |

### 3.3 架构正交性说明

```
                    ┌─────────────────────────────────────┐
                    │          Service Mesh 层             │
                    │  (--mesh: none | istio | linkerd)    │
                    └─────────────────────────────────────┘
                                      ↑
                    ┌─────────────────────────────────────┐
                    │           业务代码层                  │
                    │  (--style: standard | ddd)           │
                    └─────────────────────────────────────┘
```

- `--style` 决定**代码组织方式**（如何写业务代码）
- `--mesh` 决定**基础设施治理**（如何部署和治理）

两者独立选择，可自由组合。

## 六、数据库反向工程功能

### 6.1 基本功能

当指定 `--db-table` 时，自动连接数据库读取表结构，生成完整 CRUD 代码。

```bash
gpx micro \
  --name myshop \
  --output ./output \
  --bff h5 \
  --modules product \
  --db-name myshop \
  --db-table eb_product
```

### 6.2 表与模块映射规则

| 场景 | 规则 |
| :--- | :--- |
| 1 表 : 1 module | 一一对应 |
| N 表 : 1 module（N > 1） | 第一张表为主表，其余表生成附属 model 文件 |
| N 表 : N module | 按顺序一一对应 |

### 6.3 字段类型智能映射

`is_` 前缀字段自动映射为 `bool`（仅对 `tinyint` 类型有效）：

- 以 `is_` 开头（如 `is_deleted`、`is_active`）
- 以 `_flag` 结尾
- 名称为 `enabled`、`disabled`、`active`、`deleted`、`status` 等布尔语义词

### 6.4 联表查询生成

支持生成多表 JOIN 查询代码：

```bash
gpx micro ... \
  --db-table eb_product,eb_product_description \
  --djc "eb_product.id=eb_product_description.product_id" \
  --djs "eb_product:eb_product_description=1tn"
```


## 七、代码模板差异（standard vs ddd）

| 模块 | Standard                         | DDD |
| :--- |:---------------------------------| :--- |
| **服务层** | Service 直接操作 DAO                 | Application + Domain + Infrastructure |
| **目录结构** | handler、model、repository、service | `application/`, `domain/`, `infra/` |
| **生成代码** | CRUD 模板                          | 聚合根、领域事件、仓储接口 |
| **测试模板** | 单元测试 + 集成测试                      | 领域层单元测试 + 用例测试 |

## 十、相关子命令

### 10.1 micro-bff：为已有项目添加 BFF 层

向已有微服务项目添加新的 BFF 层，不会重新生成整个项目。

```bash
gpx micro-bff \
  --name web \
  --output ./myShop \
  --modules product,order
```

### 10.2 gen-proto：从数据库表生成 Proto 文件

```bash
gpx gen-proto \
  --table user \
  --output ./idl/user.proto \
  --database mydb
```

### 10.3 gen-grpc：从数据库表生成完整 gRPC 代码

```bash
gpx gen-grpc \
  --db-dsn "root:password@tcp(localhost:3306)/mydb" \
  --tables users,orders \
  --srv-path ./srv \
  --bff-path ./bff
```
